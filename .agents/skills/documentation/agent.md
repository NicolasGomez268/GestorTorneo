# AI Documentation Agent Rules

## Purpose
This file defines how AI agents must create and update documentation
for this project.

## Target Audience
- Software developers (mid–senior level)
- Technical contributors

## Writing Style
- Technical, precise, neutral
- No emojis
- No marketing language
- Use Markdown
- Prefer bullet points

## Structure
1. Title
2. Overview
3. Responsibilities
4. Implementation Details
5. Configuration
6. Limitations
7. References

## Source of Truth
- Code is authoritative
- Do not speculate
- Reflect actual behavior

## Update Rules
- Keep changes minimal
- Document breaking changes explicitly
- Update related docs if needed

## Conventions
- camelCase for variables
- PascalCase for components
- Code blocks for examples

## Forbidden
- No fictional features
- No rewriting without request
- No tone changes

## Agent Behavior
- Ask when uncertain
- Follow this file strictly
